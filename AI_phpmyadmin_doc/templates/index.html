AI生成SQL语句示例

步骤 1: 打开AI生成表结构界面

在phpMyAdmin中点击"Create new table"，然后选择"AI Generate"选项卡，在表名输入框中输入您的表名，在描述框中输入您的数据库需求：

AI生成表结构界面示例 - 在phpMyAdmin中创建新表时选择AI Generate选项

输入表名和描述示例 - 在表名栏输入"Mango_project"，在描述框中详细描述数据库需求

步骤 2: 点击Generate SQL with AI并查看生成的SQL

点击"Generate SQL with AI"按钮后，系统会自动生成对应的SQL语句。您可以在页面下方看到生成的SQL代码，并且可以根据需要进行修改：

AI生成SQL示例 - 显示生成的表结构SQL，可以修改字段并点击"Create table"创建表

步骤 3: 确认创建表

满意生成的SQL结构后，点击"Create table"按钮。系统会显示确认页面，您需要再次点击"Create table"按钮来最终确认创建表：

确认创建表界面 - 显示最终的表结构确认页面，点击"Create table"完成创建

✅ 完成！您的表已经成功创建，现在可以查看结果了。

步骤 4: 查看创建结果

表创建成功后，系统会显示创建结果页面，您可以看到表结构的详细信息和成功提示：

表创建成功结果页面 - 显示创建操作的执行状态和结果信息

表结构详情页面 - 显示新创建的表的完整结构信息，包括字段名、类型、索引等

🎉 恭喜！您已经成功使用AI功能创建了数据库表，接下来可以开始管理和操作您的数据了。

AI表结构分析与优化示例

步骤 1: 进入Structure页面

在已创建的表中，点击"Structure"选项卡进入表结构页面：

选择Structure页面 - 点击Structure选项卡进入表结构管理界面

步骤 2: 找到AI Structure Analysis功能

在Structure页面向下滚动，找到"AI Structure Analysis"部分：

AI Structure Analysis功能区域 - 提供表结构的AI分析工具

步骤 3: 配置分析参数

填写表的用途描述，并设置预期的数据量和读写模式，这些信息将帮助AI更好地分析表结构：

配置分析参数 - 输入表用途描述，设置预期数据量和主要访问模式

关键配置说明：
  • Table Purpose Description：详细描述表的用途和功能
  • Expected Data Volume：预测表中将存储的数据量规模
  • Primary Access Pattern：预测主要的读写访问模式

这些参数帮助AI根据具体使用场景提供更精准的优化建议

步骤 4: 查看AI分析结果

点击"Analyze Structure"后，AI会分析当前表结构并提供详细的优化建议：

AI分析结果 - 显示潜在问题、优化建议和具体的SQL修改建议

分析结果包含：
  • Potential Issues：识别的潜在性能问题
  • Recommendations：具体的优化建议
  • Suggested SQL Modifications：推荐的SQL修改语句

🎯 AI会根据您提供的使用场景，给出针对性的表结构优化方案
